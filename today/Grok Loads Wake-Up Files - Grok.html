Long self-contained Grok HTML exports are full of presentation cruft (divs, classes, Tailwind styles, avatars, timestamps, navigation, repeated headers/footers, scripts, etc.) that add zero semantic value but consume a lot of context. The repetition from Withnail debugging sessions makes it even worse. Cleaning it first will let you (and me) work with the actual content much more efficiently.

Here’s a practical, low-friction workflow tailored to your setup (Emacs + shell + tempwork habits + preference for clean Org/markdown nodes).

Recommended Primary Method: pandoc (Fastest & Highest Quality)

pandoc is excellent at turning noisy HTML into clean, semantic Markdown (or Org) while stripping most presentation-only markup.

1. Install (if you don’t have it already — one command on your Ubuntu laptop):

Bash
sudo apt update && sudo apt install pandoc

2. Basic high-quality conversion (start here):

Bash
pandoc /path/to/your-long-session.html \
  -o ~/local-side/tempwork/2026-06-05-grok-session-clean.md \
  --from html \
  --to markdown \
  --wrap=none \
  --strip-comments \
  --metadata title="Grok Session 2026-06-05 (cleaned)"

Even cleaner variants to experiment with (add these flags one at a time):

  • Strip more raw HTML and attributes:
    Bash
    -t markdown-raw_html-raw_attribute
  • Stricter / more semantic output:
    Bash
    --to markdown_strict
  • Direct to Org-mode (often nicer for you):
    Bash
    pandoc ... --to org -o ~/local-side/tempwork/2026-06-05-grok-session-clean.org

After conversion, open the .md or .org in Emacs. You’ll immediately see how much lighter and more readable it is.

Emacs-Native Alternative (No New Packages)

If you prefer to stay 100% inside Emacs:

  1. Open the HTML file.
  2. M-x eww-open-file (or just visit it) — this uses the built-in shr renderer.
  3. Use M-x shr-render-buffer on the buffer (or write a tiny helper).
  4. Or use M-x html2text if available in your Emacs.
  5. Then M-x org-mode or markdown-mode and clean further manually or with keyboard macros.

This works but pandoc is usually faster and produces more consistent structure (code blocks, lists, headings for conversation turns) with less manual cleanup.
